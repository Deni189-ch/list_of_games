Тестовое задание (Frontend, React)

Нужно создать приложение, получающее с API список игр, разбирающее и отображающее этот список. Дизайн страниц - на усмотрение разработчика.

Стэк:
React, React Router, Redux (обязательно)
CSS/SCSS, TypeScript, RTK Query (опционально)

Главная страница:

Представляет из себя список игр с обложками, и фильтр из двух селектов - Провайдер и Валюта.

Список игр может фильтроваться по провайдеру и валюте.

Сортировка игр должна производиться по полю collections.popularity в обьекте с данными игры, от меньшего к большему.

Изначально, отображается только 12 игр, следующие 12 отображаются по кнопке ”Показать еще”.

При клике на обложку игры - должен происходить переход на страницу игры, с уникальным для каждой игры URL.

Страница игры:


Представляет из себя пустую страницу, с отображенным в центре наименованием игры и кнопкой перехода на Главную страницу


Уникальный URL для каждой игры, должна открываться напрямую по этому URL.
API:
Список игр можно получить по URL: https://demo.softswiss.net/api/games/allowed_desktop
Или в файле: https://drive.google.com/file/d/1DT2FUyuqX-eXvN-2rvzaPk1akIZVUvO_/view?usp=share_link

API возвращает объект следующей структуры:


Где ключем является уникальный идентификатор игры.

В поле provider - текстовый ключ провайдера (его нужно отображать в селекте провайдера).


В поле real - карта валют, доступных для текущей игры (ключи нужно отображать в селекте валют).


В поле title - Наименование игры

Список доступных провайдеров и валют нужно получить из списка игр.

URL изображения с обложкой игры строится по следующему шаблону:
https://cdn2.softswiss.net/i/s2/${ИДЕНТИФИКАТОР}.png


Где ${ИДЕНТИФИКАТОР} - уникальный идентификатор игры (ключ в обьекте, приходящем с API}
